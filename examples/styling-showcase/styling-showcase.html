Network Topology - Styling Methods

This diagram shows a simple network topology using different styling approaches:

  • Default Styling: Top row - Server A and Client B use only the default sprotty-node CSS class
  • Subtype Styling: Second row - Database, Router, and Host get additional CSS classes based on their type
  • Custom CSS Classes: Third row left - Critical Server uses the cssClasses property for per-element styling
  • Conditional Styling: Third row right - Load Monitor changes color based on load percentage (click button to change)
  • Interactive Styling: Bottom - Gateway responds to selection and hover interactions
Try clicking and hovering on the Gateway node to see interactive styling!
