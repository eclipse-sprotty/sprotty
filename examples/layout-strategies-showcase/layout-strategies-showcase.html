Layout Strategy Comparison

Sprotty provides three main layout strategies that can be used independently or in combination:

  • Client-Only Layout: All layout computation happens in the browser. Perfect for content-heavy nodes with nested compartments, multiple sections, status indicators, and complex internal structures.
  • Server-Only Layout: Layout is computed by sophisticated algorithms (like ELK). Ideal for complex network topologies and large diagrams requiring optimal node positioning.
  • Hybrid Layout: Combines both approaches - server computes node positions while client handles internal content layout. Best for complex diagrams with rich content.
