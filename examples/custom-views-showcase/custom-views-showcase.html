Custom Views Recipe

View Patterns Demonstrated

This diagram shows various approaches to creating custom views in Sprotty:

  • Basic Custom Views: Top row - Simple geometric shapes (circle, triangle, diamond) created from scratch using SVG elements
  • Enhanced Views: Second row - Extended rectangular nodes with status indicators, borders, and decorations
  • Complex Views: Third row - Compositional views with headers, bodies, and footers demonstrating reusable view components
  • Stateful Views: Bottom row - Conditional rendering based on node state (idle, loading, success, error)
  • Custom Edges: Styled edges with different line styles (solid, dashed, dotted), colors, and animations

Interactive Features

  • Hover Effects: Move your mouse over any element to see visual feedback
  • Selection: Click on nodes to see selection highlighting with drop shadows
  • Live Updates: Watch the loading node progress animate automatically
  • Animated Edges: Some edges have moving dash patterns to show animation capabilities

Key Learning Concepts

  • TSX Syntax: How to use TypeScript + JSX for declarative SVG rendering
  • View Registry: Mapping model element types to their corresponding views
  • Rendering Context: Using context services for child rendering and view lookup
  • View Composition: Breaking complex views into manageable, reusable components
  • Conditional Rendering: Different visual representations based on model state
  • Performance: Visibility checks and efficient rendering techniques
